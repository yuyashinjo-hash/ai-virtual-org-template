# セットアップガイド

AI バーチャル組織を5分で稼働させましょう。

## 前提条件

- **Claude Code** がインストール済みで動作すること（Claude Pro / Team / Enterprise が必要です）
- プロジェクトディレクトリ（新規でも既存でもOK。テンプレートは既存コードに影響しません）

## ステップ 1: テンプレートをコピー

`.company/` フォルダ全体をプロジェクトのルートにコピーしてください：

```
your-project/
├── .company/          <-- ここにコピー
│   ├── CLAUDE.md
│   ├── secretary/
│   ├── engineering/
│   └── research/
├── 既存のファイル...
```

## ステップ 2: プロフィールを編集

`.company/CLAUDE.md` を開き、**Owner Profile** セクションを記入します：

```markdown
## Owner Profile

- **Name**: 山田太郎
- **Business / Activity**: プロジェクト管理ツールのSaaSスタートアップ
- **Goals**: Q3までにMVPリリース、顧客オンボーディングの自動化、手作業の削減
- **Created**: 2026-04-09
```

このプロフィールにより、秘書や各部門があなたの状況に合わせた対応をしてくれるようになります。

## ステップ 3: 部署をカスタマイズ（任意）

テンプレートには3つの部署が含まれています。以下のように使い分けてください：

### そのまま使う
デフォルト設定は、ほとんどの個人事業主やスタートアップ創業者、小規模チームに適しています。

### 既存の部署を変更する
各部署の `CLAUDE.md` を編集して調整できます：
- **トーン** -- よりフォーマルに、カジュアルに、あるいは自社の文化に合わせて
- **仮想メンバー** -- 名前や視点の変更、専門知識の追加
- **ルール** -- 部署固有の規約を追加

### 新しい部署を追加する
1. 新しいフォルダを作成：`.company/your-department/`
2. `CLAUDE.md` に役割・ルール・フォルダ構成を記載
3. `.company/CLAUDE.md` の部署一覧に登録
4. 必要に応じてサブフォルダを作成（例：`docs/`、`reports/`）

**追加する部署の例：**
- `marketing/` -- コンテンツカレンダー、ブランドボイスの指針、キャンペーン管理
- `finance/` -- 予算管理、請求書管理、経費区分
- `sales/` -- パイプライン管理、顧客メモ、提案書テンプレート
- `legal/` -- 契約レビュー、コンプライアンスチェック、規程文書

## ステップ 4: 外部連携を追加（任意）

秘書部門はオプションで外部ツールとの連携に対応しています。`.company/secretary/CLAUDE.md` を編集して追加してください：

```markdown
### Optional: External Integrations
- **Task database**: Notion DB `your-database-id-here`
- **Calendar**: Google Calendar `your-calendar-id@group.calendar.google.com`
```

## ステップ 5: 使い始める

プロジェクトディレクトリで Claude Code を起動し、次のように伝えてください：

> 「.company/CLAUDE.md を読んで、バーチャル組織として動いてください。」

これ以降、秘書があなたの主要な窓口になります。以下のような指示を試してみてください：

- **「今日何をすべき？」** -- 秘書がTODOと優先事項を確認します
- **「新しいAPIを設計したい。相談させて。」** -- エンジニアリングチームがアプローチを議論します
- **「ヨーロッパ展開のアイデアがある。」** -- 受信箱に記録され、リサーチ部門が分析します
- **「TODO追加：金曜までにオンボーディングフローを完成、優先度高」** -- タスクが記録されます
- **「今日はここまでにしよう。」** -- セッション終了ルーティン：決定の記録、TODO更新、引き継ぎ準備

## より効果的に使うためのコツ

1. **ルーティンを定着させる** -- セッションの開始と終了をルーティン化しましょう。文脈は時間とともに蓄積されていきます。
2. **記録はお任せ** -- わざわざメモを取らなくて大丈夫です。決定事項やアイデアはシステムが自動的に記録します。
3. **受信箱を活用** -- どこにも分類できない思いつきは、そのまま伝えてください。秘書が受信箱に入れ、後で整理します。
4. **週次レビュー** -- 週に一度、`secretary/notes/` と `research/proposals/` を確認し、記録された内容や提案を振り返りましょう。
5. **定期的にコミット** -- `.company/` フォルダはバージョン管理下に置くのがおすすめです。セッション後にコミットすれば履歴が残ります。

## フォルダ構成リファレンス

```
.company/
├── CLAUDE.md                          # 組織設定（最初に編集）
├── secretary/
│   ├── CLAUDE.md                      # 秘書部門の設定
│   ├── inbox/                         # 未整理のアイデア・メモ
│   │   └── YYYY-MM-DD.md             # 日次の受信箱
│   ├── todos/                         # 日次タスクファイル
│   │   └── YYYY-MM-DD.md             # 今日のタスクリスト
│   └── notes/                         # 決定ログ・メモ
│       ├── YYYY-MM-DD-decisions.md    # その日の決定事項
│       ├── YYYY-MM-DD-learnings.md    # その日の学び
│       └── topic-name.md             # トピック別メモ
├── engineering/
│   ├── CLAUDE.md                      # エンジニアリング部門の設定
│   ├── docs/                          # 技術ドキュメント・設計書
│   └── debug-log/                     # バグ調査ログ
└── research/
    ├── CLAUDE.md                      # リサーチ部門の設定
    ├── proposals/                     # リサーチ提案
    ├── rejected/                      # 却下されたアイデア（繰り返さない）
    └── watchlist/                     # 保留中のアイデア（後日再検討）
```
